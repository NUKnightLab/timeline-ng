Timeline data format

The JSON shape behind every Timeline NG timeline, whether it's saved to your PDS or exported as a file.

Overview

A timeline is a single JSON object with an optional title slide and an array of events. If you're writing timelines by hand or generating them from code, this page describes every field. If you want a machine-checkable version, see the JSON Schema section further down this page. 

{
  "title": { "text": { "headline": "My Timeline" } },
  "events": [
    {
      "start_date": { "year": 2020, "month": 1, "day": 1 },
      "text": { "headline": "First event", "text": "<p>What happened.</p>" },
      "media": { "url": "https://example.com/photo.jpg", "caption": "A photo" }
    }
  ]
}

Timeline

FieldTypeNotes
titleEventOptional. Shown first, without a date, as an intro slide.
eventsEvent[]Required. Rendered in chronological order.
settingsSettingsOptional. Playback and display configuration — see below.

Settings

FieldTypeNotes
languagestringLocale code for UI chrome and date formatting. "en" and "es" have full UI translations; other codes fall back to English chrome but still format dates correctly. Default: "en".
theme"light" | "dark" | "auto"Color theme. "auto" follows the viewer's OS/browser preference. Default: "auto".
initialIndexnumber0-based index of the slide shown first (counting the title slide, if present). Default: 0.
reverseOrderbooleanShow events newest-first. The title slide, if any, always stays first. Default: false.

Event

FieldTypeNotes
unique_idstringOptional identifier, useful for deep-linking to a specific slide.
start_dateDateRequired on every event except the title slide.
end_dateDateOptional. Renders the event as a date range.
textTextHeadline and body copy.
mediaMediaAn image, video, audio, or embed for the slide.
media_position"left" | "right"Which side the media appears on. Defaults to "left".
groupstringOptional label; events sharing a group are visually clustered in the navigation.
backgroundBackgroundOptional color or image behind the slide.

Date

Only year is required — add more fields for finer precision.

FieldTypeNotes
yearnumberRequired. Negative values are BCE. Very large magnitudes (cosmological scale) are supported.
monthnumber1-based (January is 1).
day, hour, minute, second, millisecondnumberAdd as much precision as you have.
display_datestringOverride the rendered date text entirely, e.g. "circa 1900".

Text

FieldTypeNotes
headlinestringMay contain inline HTML (bold, italic, links).
textstringBody copy. May contain HTML.

Media

FieldTypeNotes
urlstringA media URL — image, video, audio file, or a link the player knows how to embed (YouTube, Spotify, Bluesky posts, and more — see the media types guide).
captionstringShown beneath the media.
creditstringAttribution text.
altstringAlt text for images. Falls back to caption if omitted.
thumbnailstringOptional thumbnail override for the navigation track.

Media saved through the editor's Atmosphere browser also carries a blobRef field pointing at a PDS-hosted blob — you won't need to set this by hand.

Background

FieldTypeNotes
colorstringAny valid CSS color.
urlstringA background image URL.

JSON Schema

If you're generating or validating timeline JSON programmatically — especially outside JavaScript — the @knight-lab/timeline-ng-core npm package ships a JSON Schema for this exact format at schema/timeline.schema.json, generated directly from the TypeScript types so it can't drift out of sync. See the package README for a validation example.

Migrating from TimelineJS 3

If you have existing TimelineJS 3 JSON or a Google Sheets CSV export, the @knight-lab/timeline-ng-core package's fromTL3 and fromTL3CSV functions convert them to this format.

You could also do this in the editor if that's easier!
