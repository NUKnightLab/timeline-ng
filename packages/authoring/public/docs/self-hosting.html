Self-hosting your timeline

Embed a timeline on your own site without using the Atmosphere network.

Overview

The Timeline NG player is a standalone web component you can include on your own web pages. You can export your timeline data as a JSON file from the editor, host it alongside your site, and point the player at it. No server-side code required. You do have to use modern javascript build tools; unlike our other tools, there is currently no support for simply loading Timeline NG with a <script src="..."> tag.

1. Export your data

In the editor, open Publish and click Download JSON. Save the file somewhere in your site's assets — for example timeline.json. See the timeline data format guide if you're writing or generating this JSON by hand instead.

2. Install the player

Install via npm:

npm install @knight-lab/timeline-ng

Then import the styles once in your app entry point:

import '@knight-lab/timeline-ng/styles.css';

3. Add the player to your page

The player ships as a Svelte component. Use loadTimeline to fetch your JSON, then pass the result to SlidePlayer: 

import { SlidePlayer, loadTimeline } from '@knight-lab/timeline-ng';
import '@knight-lab/timeline-ng/styles.css';

const result = await loadTimeline('/path/to/timeline.json');
if (result.ok) {
  // mount SlidePlayer with result.timeline
}

See the project README for complete framework integration examples.

4. Configuration options

If your timeline JSON includes a settings object (see the data format guide), the Timeline NG embed app reads it automatically and applies it. When using SlidePlayer directly, as shown above, you're responsible for reading timeline.settings yourself and passing the matching props — the props below let you override or supply these values manually:

  • language — locale code for date formatting (default: "en")
  • theme — "light", "dark", or "auto" (default)
  • initialIndex — slide number to start on (default: 0)
  • reverseOrder — show events newest-first (default: false)

5. Styling

The player exposes CSS custom properties for theming. Set them on the container or any ancestor element: 

#my-timeline {
  --tl-color-accent: #0066cc;
  --tl-font-heading: 'Georgia', serif;
  --tl-stage-min-height: 300px;
}

Example apps

The timeline-ng repository includes a few small example apps demonstrating different integration patterns:

  • example-svelte5
    Fetches a timeline at runtime (from an AT URI or any URL) with loadTimeline() inside a Svelte component.
  • example-vanilla
    Mounts the player with a plain, hand-written timeline object — no Svelte component authoring required.
  • example-svelte5-json
    Renders the player from a static JSON file bundled with the app — the pattern this guide walks through above.
  • example-custom-theme
    Demonstrates taking more control over the visual style of a timeline and switching between languages.
